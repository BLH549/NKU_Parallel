| 指令集 | 位宽 | 并行度 | 说明 |
|--------|------|--------|------|

## 三、MD5 核心计算（需 SIMD 化）

### F/G/H/I 四个非线性函数

```c
F(x,y,z) = (x & y) | (~x & z)
G(x,y,z) = (x & z) | (y & ~z)
H(x,y,z) = x ^ y ^ z
I(x,y,z) = y ^ (x | ~z)
```

### FF 宏（64轮中第1-16轮）

```c
#define FF(a,b,c,d,x,s,ac) { \
  (a) += F(b,c,d) + x + ac; \
  (a) = ROTATELEFT(a,s) + b; \
}
```

---

## 四、NEON 并行化策略

**核心思路**：用 128-bit 寄存器同时处理 **4 条消息**

```
Q0 = [msg1_A, msg2_A, msg3_A, msg4_A]  ← 4条消息的 A 值
Q1 = [msg1_B, msg2_B, msg3_B, msg4_B]  ← 4条消息的 B 值
...
```

### 关键实现细节

1. **填充阶段**：每条消息独立填充，无法 SIMD
2. **64轮迭代**：每轮用 NEON 并行处理 4 条消息
3. **消息字加载**：`vld1.32` 一次性加载 4 个消息的 x[i]
4. **常量 T[i]**：`vdup.32` 复制到所有通道

### 2. O1/O2 级别：NEON **有加速**

| | O1 | 加速比 | O2 | 加速比 |
|--|------|-------|------|-------|
| Baseline Hash | 3.00s | - | 2.86s | - |
| NEON Hash | 1.82s | **1.65x** | 1.71s | **1.67x** |

**原因**：
- 编译器优化后，NEON 的**数据级并行**优势体现
- 64 轮迭代的 F/G/H/I 运算被高效并行执行

| 因素 | 说明 |
|------|------|
| **并行度只有 4** | 128-bit NEON 每次处理 4 条消息，并行度不高 |
| **消息长度不同** | 不同口令填充后块数不同，需要回退到逐个处理 |
| **填充阶段串行** | `StringProcess()` 填充无法 SIMD 化 |
| **内存访问瓶颈** | 数据在 DRAM 和寄存器间移动耗时 |

### 5. 进一步优化建议

1. **提高批量大小**：用 AVX2（256-bit/8路）或 AVX-512（512-bit/16路）
2. **预填充缓冲区**：避免每批都重新分配内存
3. **按消息长度分组**：减少回退到串行处理的情况
